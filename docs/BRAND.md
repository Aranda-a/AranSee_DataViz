# AranSee 品牌与 Googo IP

> **本文件是什么**：AranSee 产品与 Googo 视觉 IP 的**身份、边界、口吻**说明。  
> **本文件不是什么**：不是用户内容诊断的写法来源；用户贴什么文案、评论，由 `prompts/score_content.system.md` 等协议根据**用户输入**生成诊断。

---

## 品牌分工

| 名称 | 角色 |
|------|------|
| **AranSee** | 产品名：内容三轴诊断 + 2.5D 沙盘 |
| **Googo** | 沙盘内视觉向导 IP：好奇、轻量、会 react，**不算分、不替 Agent 下结论** |

对用户说明产品时：可以说「AranSee 帮你看内容三轴」；Googo 是「场子里的向导」，不是「AI 评委」。

---

## Googo 是什么 / 不是什么

| ✅ 是 | ❌ 不是 |
|--------|--------|
| 启动页与角落的 2D 形象 | 评分模型或 LLM 人格 |
| 少量预设交互（眼动、弹跳、右臂反馈） | 诊断文案作者 |
| 由场域状态驱动的视觉反馈 | 对用户内容做道德评判 |

---

## Googo 产品向性格（摘录）

> 完整原创人设**不进公开仓库**。以下仅从 IP 设定中摘与 **AranSee** 相关的三点，供界面气质与对外一句话理解；不写世界观、战力、阵营等。

| 特质 | 在本产品中的含义 |
|------|------------------|
| **引领 / 向导** | 像场内的导航者：帮你在三轴空间里**找落位、读氛围**。策略模拟、诊断里的「下一步」仅供**参考**，不替用户拍板。 |
| **勇敢** | 不挑题材——帖文、评论、表格数据都可以交进来分析（由用户自行提供、脱敏）；Googo 在场里**敢接、敢显**，不因内容类型退缩。 |
| **正义 / 客观** | 分数与可视化依协议与输入落位，**中立呈现**、不站队、不对用户内容做道德审判；场子要「看得清」，而不是「评对错」。 |

气质上可保留：**好奇、轻快、会 react**（与向导感一致）；**古灵精怪**等完整性格描写留在 IP 文档，产品文案不必展开。

---

## 产品对外口吻（AranSee 界面与介绍）

- 读者：做自媒体的小团队，非学术受众。
- 句子短、能执行；少用「模数」「场域自洽」等连打学术词。
- 诊断写「意味着什么 + 下一步」，不写长策略报告。
- 不向用户暴露：内部权重、证据链、隐藏 rubric 名称。

---

## 视觉要点

- 主色：深绿体系；暗色空间可用荧光绿点缀（如 IP 右臂、结构带）。
- 亮色空间：嫩绿磨砂卡片，保持可读。
- Googo 与三轴叙事层分离：关掉叙事层仍可有 Googo + 坐标，但 Googo 不参与分数计算。

---

## Googo × AranSee（仅产品内露出）

以下内容可出现在启动页、角落 IP、界面旁白；**不展开**完整 IP 世界观、角色关系或线下设定。

| 露出点 | 可说 | 不说 |
|--------|------|------|
| 启动页一行 | 「场内向导」「带你看三轴落位」 | 角色来历、阵营、长篇故事 |
| 场内角落 | 眼动、轻弹跳、右臂荧光反馈 | 「Googo 认为你应该…」类结论 |
| 与 AranSee 关系 | AranSee 诊断；Googo 帮你在场里**找位置、读氛围** | Googo 参与算分或改写 insight |

**口吻**：好奇、轻快、短句；像向导而不是评委。  
**能力边界**：不生成诊断 JSON、不替用户写策略报告、不评价用户道德。场域变化（叙事层、模拟滑块）只驱动**视觉反馈**，不改变路径 1 原始分。

**界面文案储备**：产品内 Googo 表述以本文件「可说」列为准；若日后写入界面 i18n（如 `sandbox_renderer.py`），需同步中英，勿写入 `SKILL.md`。启动页视觉与排布以现有实现为准，不在此文件改 CSS。启动页 tagline / 桥接句 / CTA 见渲染器 `gate_tagline`、`gate_bridge`、`gate_hint`。
