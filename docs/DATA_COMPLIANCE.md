# 数据与合规

面向公开仓库、展示案例与 Skill 使用者。

---

## 数据原则（BYOD）

1. AranSee **不提供**平台爬取、批量抓取、未授权 API 工具。  
2. 分析输入由**用户自行提供**（粘贴帖文、自写摘要、手动抽样的脱敏评论）。  
3. 公开案例仅含**脱敏样本**；不含可复原链接、视频文件、全量评论库。  
4. 案例说明为「公开页手动抽样 + 抽象化」，**不代表全量统计结论**。

**README 硬边界（建议保留）：**

> 请仅分析你有权使用的文本。AranSee 是诊断工具，不鼓励、也不内置任何绕过平台规则的数据获取方式。

---

## 与抄袭 / 追责的关系

| 对象 | 风险 | 建议处理 |
|------|------|----------|
| 他人帖文 / 评论 | 可识别转载可能触及版权或平台规则 | 脱敏、摘要代替原文、小样本；不含 BV / 昵称 / 真名 |
| 他人作品（影视解说等） | 分析结构 ≠ 二次发行内容 | 强调**诊断与结构判断**；公开包**不出现创作者真名、片单可识别组合** |
| AranSee 方法论 | 轴定义、映射、Googo、prompt 结构被照搬 | 代码 MIT；**内核 rubric / 权重闭源**；商标与 IP 文案保留 |

**不算抄袭的部分：** 你对内容的结构化判断（三轴、场域类型、策略含义）属于原创分析框架。  
**需要小心的部分：** 素材是否合法取得、公开案例是否**过度可识别**（真名 + 真片名 + 可搜组合）——属数据合规，不是「做了 AI 分析」本身。

## 展示案例特别声明

公开仓库中的 `assets/cases/` 与 `demos/`：

- 是**脱敏后的样例场**，用于演示 AranSee 协议与沙盘形态；  
- **不是**训练数据包，不授权用于模型训练或二次数据集分发；  
- 不含创作者真名、账号链接、可一键复原的源视频信息；  
- 若你识别出与某位创作者相似，纯属题材类型接近，**不代表官方合作或授权**。

如发现公开包仍有可识别隐私信息，请通过 Issue 联系下架或进一步脱敏。
